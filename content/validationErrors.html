Validation Errors

This page will provide explanation and fixes for most commonly found Validation errors

What are Schema Errors? How do they occur?

Schema errors occur due to incorrect file layout Most commonly found file format errors are
1) Headers have Spelling issues - Headers have to exactly match how its displayed in the File Spec
2) Incorrect column order - The Order of columns should be as listed in the file spec. Any deviation will generate an error
3)Extra Double Quotes - If data itself may have double quotes, they need to be escaped or removed before providing the data, otherwise the layout of the file will be compromised
4)Incorrect Data Formats - ALL Dates must be converted to character strings. Date formats for each date-related attribute are indicated in the data dictionary and it should be in format of YYYY-MM-DD
5)Incorrect Time formats - Time must be in 24 hour format (HH:MM)
5)Incorrect Data Type - Mismatch found for Type, Format, Nullability of a column. Eg: if a required field like Person_id is passed as a null value. This will be a cause for rejection of the file

What are data integrity errors?

Data Integrity occurs primarily due to these issues
1) Refrential Integrity Issue - If an uploaded file has Id which is dependent on a master file for look up eg: STG_SIH_CLASSENROL_ATTEN.STUDENT_ID should be existing in STG_SIH_PERSON.PERSON_ID
2) Primary key duplicates - When a rule (one or more combination of columns) for uniqueness is broken
