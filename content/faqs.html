Frequenty Asked Questions

What do I start?

Before you begin programming/development to extract the data, please make sure you have completed a discovery/knowledge session with an IBM or Onboarding Data Specialist. We have come up with a logical order to enable frequent playbacks before go live. Click link Overview and review the Table provided. The Table provides the order of data to be uploaded. eg: 'Login' Module is required before moving to 'Class Cards' module. The 'Login' section provides the categories of data required. The page also provides a detail view of file priority
How is STG_SIH_CLASS_SCHEDULE.Class_Id to be populated?

STG_SIH_CLASS_SCHEDULE.Class_Id is a unique identifier to identify a Course Section. A Course Section can span a year or a term. For each unique Class_Id the Application will break out a Class_Card. If the district does not want to break out class cards by semester for a Course, a unique Class_id needs to be provided across terms.
How to define class card title?

We reccomend to populate STG_SIH_CLASS_SCHEDULE.COURSE_SECTION_TITLE with a value which will distinctively identify a class and also allow a meanigful sort/group. Also consider what the teachers are used to seeing as Class names in their SIS, Gradebook or LMS system. The default sort order is alphabetical. A reccomended nomenclature -> Period + ' '+ Course Title+ ' ' + Section + ' ' + Term
What if start and end time for a class is not entered in the System by the teacher?

In this scenario we would ask the Data Admin to put in a place holder time and communicate to teachers to put in the actual start and end time
What Days do I need to populate SCHOOL_CALENDAR?

By School, all days should have a unique record including Holidays, Weekends and Off days.
Will Watson upload a file if some of the records have Schema issues?

No. The file cannot be loaded unless all the records pass the schema validation test.
Are all the files required for the Application to Work?

No. While we strongly reccomend all files to be provided, we understand the limitations of data availability either due to Privacy concerns or lack of data in certain areas. The system will work if non-essential files are not provided. This needs to be reviewed and approved by the stakeholders.
Are all the files required for running the validation?

No. The district can provide a partial list of files. The validation will call out the missing files. Please note this is applicable only during the initial onboarding phase.
If the district does not have data for a non-essential file what do we do?

In this scenario, the district needs to provide an empty file with the row headers.
Our District does not have Roles like 'Superitendent', 'Teachers' and 'Principal' in system. If we send the Job Role will Watson be able to identify the Application roles?

Yes to a certain extent. We do parse the Job Desc to identify the Application Role. Eg: 'Middle School Principal' will be assigned a 'Principal' Role whereas 'SpEd Co-ordinator' may not be mapped. We reccomend that District applies the rules to provide the application roles for optimal behavior.
