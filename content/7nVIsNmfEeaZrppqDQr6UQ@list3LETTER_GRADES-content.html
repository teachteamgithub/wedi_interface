Table @list:LETTER_GRADES

DescriptionThe implementation team collects the letter grade ranges from the district. If the district has grade ranges defined only at the district level, the organization id will contain a character string representing no data '*', we do not use null. The district can also opt to define school specific ranges.

A couple of special rows must also be created to handle non-numeric elementary grades and cases where no grade is available.
A row with a range of -1 -1 with a letter grade ‘No grade given’ This would be used where the grade is null or blank.
A row with a range of -2 -2 with a letter grade ‘Satisfactory’
A row with a range of -3 -3 with a letter grade ‘Unsatisfactory’

The interval ranges should be collected to 2 decimals. For example, a B is 60.00 to 74.99.

Part of the implementation process should be to validate the text a district's for "satisfactory" and "unsatisfactory" elementary results. If they use something other than Satisfactory the label can be changed, but the range must stay the same. If the district uses more than 2 descriptors for text results they should be collected and the data warehouse team must be contacted so DataStage jobs that do look ups to the letter grade dimension can be changed.
